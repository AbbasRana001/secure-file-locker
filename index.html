History & Theory

The Vigenère
Cipher

First described by Giovan Battista Bellaso in 1553 and later misattributed to Blaise de Vigenère, this polyalphabetic cipher was considered unbreakable for over three centuries — earning it the name "le chiffre indéchiffrable."

What is it?

The Vigenère cipher is a method of encrypting text by applying a series of different Caesar ciphers, based on the letters of a repeating keyword. Unlike the Caesar cipher — which shifts every letter by a single fixed number — the Vigenère cipher shifts each letter by a different amount, making it far more resistant to basic analysis.

Each letter of the keyword maps to a shift value (A = 0, B = 1, … Z = 25). The keyword repeats along the entire message, pairing each plaintext character with a key character that defines how far to shift it.

Step-by-step breakdown

1

Map letters to numbers

Convert each letter to a value: A = 0, B = 1 … Z = 25. Non-letters (spaces, punctuation, digits) are kept unchanged.

2

Repeat the keyword

Cycle your keyword over the full message so every plaintext letter has a matching key letter aligned beneath it.

3

Shift and wrap at 26

Add (encrypt) or subtract (decrypt) the key value from the letter value. If the result goes outside 0–25, wrap it using modulo 26.

The formulas

Encrypt E(i) = ( P(i) + K(i) ) mod 26 Add key shift, wrap at 26
Decrypt D(i) = ( C(i) − K(i) + 26 ) mod 26 Subtract shift, wrap at 26

P = plaintext value  •  C = ciphertext value  •  K = key value  •  all in range 0–25

Strengths and limitations

Beats frequency analysis

The shifting key means each letter is encoded differently every time it appears, hiding the letter-frequency patterns that break simple Caesar ciphers.

Symmetric encryption

The same keyword encrypts and decrypts. Both parties must share the key in advance through a secure, separate channel.

Key length matters

A longer keyword reduces periodicity, making statistical attacks significantly harder. A key as long as the message (used once) becomes a one-time pad.

Not modern-secure

The Kasiski examination can recover short repeating key lengths, breaking the cipher. It is not suitable for real-world security applications today.
