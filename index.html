History & Theory

The Vigenère Cipher

First described by Giovan Battista Bellaso in 1553 and later misattributed to Blaise de Vigenère, this polyalphabetic cipher was considered unbreakable for over three centuries — earning the name "le chiffre indéchiffrable."

What is it?

The Vigenère cipher encrypts text by applying a series of Caesar ciphers based on the letters of a repeating keyword. Unlike a Caesar cipher — which shifts every letter by one fixed amount — Vigenère shifts each letter by a different amount, which defeats simple frequency analysis.

Each letter of the keyword maps to a shift value: A = 0, B = 1, … Z = 25. The keyword repeats across the full message, pairing each plaintext character with a key character that determines how far to shift it.

Step-by-step

1

Map letters to numbers

A = 0, B = 1 … Z = 25. Spaces, digits, and punctuation pass through unchanged.

2

Repeat the keyword

Cycle your keyword across the full message so every letter has a matching key letter.

3

Shift and wrap at 26

Add (encrypt) or subtract (decrypt) the key value. Wrap results with modulo 26.

The formulas

EncryptE(i) = ( P(i) + K(i) ) mod 26Add key shift, wrap at 26
DecryptD(i) = ( C(i) − K(i) + 26 ) mod 26Subtract shift, wrap at 26

P = plaintext value  ·  C = ciphertext value  ·  K = key value  ·  all in range 0–25

Strengths & limitations

🛡

Beats frequency analysis

The shifting key encodes the same letter differently each time, hiding the patterns that break Caesar ciphers.

🔑

Symmetric encryption

The same keyword encrypts and decrypts. Both parties must share it through a separate secure channel.

📏

Key length matters

Longer keywords reduce periodicity. A key as long as the message, used once, is equivalent to a one-time pad.

⚠️

Not modern-secure

The Kasiski examination can recover short repeating keys. Do not use this for real-world sensitive data.
